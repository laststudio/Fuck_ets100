# 一、云端模式概览

云端模式是一种**在线获取作业答案**的工作方式，与本地读取模式（Shizuku / Root / 零宽字符直读）不同：

| 对比维度 | 本地读取模式 | 云端模式 |
|----------|-------------|---------|
| 数据来源 | 手机本地 `Android/data/com.ets100.secondary/` | ETS100 云端 API + CDN 资源包 |
| 权限需求 | 全文件访问 / 悬浮窗 / 应用列表 | **无需特殊系统权限** |
| 账号要求 | 不需要 | 需要 ETS100 手机号 + 密码 |
| 顶号风险 | 无 | **会顶掉 ETS100 官方客户端登录** |
| 网络依赖 | 离线可用 | 必须联网 |
| 数据完整性 | 取决于本地已下载资源 | 取决于云端作业列表 |

**核心思路**：使用 ETS100 账号登录云端接口 → 获取作业列表 → 从 CDN 下载 ZIP 资源包 → 本地解压并解析 `content.json` → 提取题目与答案。

# 三、完整工作流程

```
Step 1: 用户进入 设置 → 运行授权 → 选择"云端模式"
    │
    ▼
Step 2: 点击"登录 ETS100 账号" → 进入 CloudActivationScreen
    │  输入手机号 + 密码
    │
    ▼
Step 3: 调用 POST /user/login
    │  ├─ 成功 → 获取 token → 继续
    │  └─ code=30014 → 设备需要绑定 → 自动调用 POST /user/rebind-code
    │
    ▼
Step 4: 用 token 调用 POST /m/ecard/list → 获取 parent_account_id
    │
    ▼
Step 5: 保存认证信息到 SharedPreferences
    │  (token, phone, password, parent_account_id, device_code)
    │
    ▼
Step 6: 回到读取页，点击右下角"读取"按钮
    │  → 弹出风险提示对话框 → 用户确认
    │
    ▼
Step 7: loadCloudHomeworkList()
    │  ├─ 用保存的手机号+密码自动重新登录（刷新 token）
    │  ├─ 调用 POST /g/homework/list → 获取作业列表
    │  └─ 解析 base_url + data[] → 展示作业列表
    │
    ▼
Step 8: 用户点击某个作业 → downloadAndParseHomework()
    │  ├─ 拼接 CDN URL: base_url + content.url
    │  ├─ 下载 ZIP → cacheDir/cloud_homework/
    │  ├─ 验证 ZIP Magic Number
    │  ├─ ZipPasswordGenerator 生成 64 字符密码
    │  ├─ zip4j 解压（带密码）
    │  ├─ 读取 content.json → parseContentJson()
    │  └─ 构建 Paper 对象 → 存入 downloadedPapers
    │
    ▼
Step 9: 展示答案详情
```

# 四、认证机制

## 4.1 全局配置

```kotlin
API_BASE_URL = "https://api.ets100.com"
CDN_BASE_URL = "https://cdn.subject.ets100.com"
PID          = "grlx"
SECRET_KEY   = "555ffbe95ccf4e9535a110170b445ab8"
```

## 4.2 机器码生成

Android 版本**不读取真实硬件信息**，而是首次运行时随机生成伪机器码并持久化到 SharedPreferences，保证后续一致性：

```
首次生成：
  randomHex(16) → MD5 → 取 [8:24] 作为 dataMd5（16字符）
  randomHex(16) → MD5 → 取 [8:24] 作为 macMd5 （16字符）
  机器码 = dataMd5 + "|" + macMd5  （共 33 字符）

后续使用：直接从 SharedPreferences 读取已保存的机器码
```

> **注意**：Python 桌面版从 Windows 注册表读取 `ProductId` + `MachineGuid` + `ETSKey\InstallDate` + MAC 地址，再计算 MD5。Android 版做了简化，使用随机值替代。

## 4.3 登录流程

```
POST /user/login

请求参数：
  sn                = "test"
  phone             = 手机号
  password          = 密码（明文）
  device_code       = 机器码
  device_name       = "DESKTOP"
  version           = "3"
  local_ip          = "127.0.0.1"
  system            = "4"
  global_client_version = "5.4.5"
  sign_response     = 1

响应：
  正常 → { "body": { "token": "xxx" } }
  code=30014 → 设备需要绑定，抛出 DeviceBindRequiredException
```

## 4.4 设备绑定

当登录返回 `code=30014` 时，自动调用绑定接口：

```
POST /user/rebind-code

请求参数：
  sn            = "test"
  phone         = 手机号
  password      = 密码（明文）
  code          = "0"（固定值）
  version       = "2"（注意：绑定使用 version=2，与登录不同）
  device_name   = "1337"（固定值，与登录的 "DESKTOP" 不同）
  device_code   = 机器码
  local_ip      = "127.0.0.1"
  system        = "4"
  global_client_version = "5.4.5"
  sign_response = 1

响应：
  成功 → { "body": { "token": "xxx" } }
  绑定成功后直接返回 token，无需再次登录
```

## 4.5 获取父账户 ID

```
POST /m/ecard/list

请求参数：
  sn        = "test"
  token     = 登录返回的 token
  version   = "3"
  system    = "4"
  global_client_version = "5.4.5"
  sign_response = 1

响应：
  {
    "body": {
      "0": {
        "parent_id": "123456"
      }
    }
  }
```

解析逻辑：取 `body` 对象的第一个 key（通常是 `"0"`），从中读取 `parent_id`。

## 4.6 Token 自动刷新

在每次加载作业列表前，**自动使用保存的手机号 + 密码重新登录**以获取最新 Token：

```kotlin
// ReadScreen.kt - loadCloudHomeworkList()
val phone = ETS100AuthManager.getPhone(context)
val savedPassword = ETS100AuthManager.getPassword(context)
val deviceCode = ETS100AuthManager.getDeviceCode(context)
val loginResult = ETS100ApiClient.login(phone, savedPassword, deviceCode)
// → 获取新 token → 保存到 AuthManager → 用于后续请求
```

这样做的原因是：Token 可能过期，而密码以明文保存，可以随时重新登录获取新 Token。

## 4.7 认证信息持久化

使用 SharedPreferences（文件名 `ets100_auth`）存储：

| Key | 说明 | 登出时 |
|-----|------|--------|
| `device_code` | 机器码（首次随机生成） | **保留** |
| `phone` | 登录手机号 | 清除 |
| `password` | 登录密码（明文） | 清除 |
| `token` | API Token | 清除 |
| `parent_account_id` | 父账户 ID | 清除 |
| `is_logged_in` | 登录状态标记 | 清除 |

> **注意**：登出时保留 `device_code`，确保同一设备始终使用相同机器码，避免触发设备绑定验证。

# 七、资源下载逻辑

## 7.1 URL 拼接规则

```kotlin
val zipUrl = if (content.url.startsWith("http")) {
    // 绝对 URL，强制 HTTPS
    content.url.replaceFirst("http://", "https://")
} else {
    // 相对路径，拼接 base_url
    var baseUrl = cloudBaseUrl
    if (baseUrl.startsWith("http://")) {
        baseUrl = baseUrl.replaceFirst("http://", "https://")
    }
    val fullUrl = if (content.url.startsWith("/")) content.url else "/${content.url}"
    baseUrl.trimEnd('/') + fullUrl
}
```

示例：
```
base_url = "https://cdn.subject.ets100.com"
url      = "/resource/abc123.zip"
→ 完整URL = "https://cdn.subject.ets100.com/resource/abc123.zip"
```

## 7.2 下载请求头

```
Host: cdn.subject.ets100.com
User-Agent: libcurl-agent/1.0
Accept: */*
```

## 7.3 HTTPS 证书处理

CDN 域名 `cdn.subject.ets100.com` 的 SSL 证书配置存在问题（证书实际签发给 `default.chinanetcenter.com`），因此下载时**跳过了 SSL 证书验证**：

```kotlin
// 信任所有证书的 TrustManager
val trustAllCerts = arrayOf(object : X509TrustManager {
    override fun checkClientTrusted(chain: Array<X509Certificate>, authType: String) {}
    override fun checkServerTrusted(chain: Array<X509Certificate>, authType: String) {}
    override fun getAcceptedIssuers(): Array<X509Certificate> = arrayOf()
})
val sslContext = SSLContext.getInstance("TLS")
sslContext.init(null, trustAllCerts, SecureRandom())
connection.sslSocketFactory = sslContext.socketFactory
connection.hostnameVerifier = HostnameVerifier { _, _ -> true }
```

## 7.4 缓存路径

```
context.cacheDir/cloud_homework/
├── abc123.zip              ← 下载的 ZIP 文件
```

# 八、ZIP 密码生成与解压

## 8.1 ZIP 文件验证

下载完成后，先读取文件头 4 字节验证 Magic Number：

| Magic Number | 说明 |
|-------------|------|
| `504B0304` | 标准 ZIP 文件 |
| `504B0506` | 空 ZIP（无数据） |
| `504B0708` | ZIP64 格式 |

如果 Magic Number 不匹配，则跳过该文件。

## 8.2 密码生成算法

```kotlin
// ZipPasswordGenerator.kt

// 1. 读取 ZIP 文件尾部 336 字节
val footer = readLastBytes(zipFile, 336)

// 2. 验证文件签名
//    签名位置 1: footer[0:8]   == "MSTCHINA"
//    签名位置 2: footer[144:149] == "EPLAT"
val signatureValid = footer[0..7] == "MSTCHINA" || footer[144..148] == "EPLAT"

// 3. 提取 128 字节种子数据（偏移 16-143）
val seed = footer[16..143]

// 4. 第一次 MD5 → 32 字符 HEX（大写）
val firstMd5 = MD5(seed).hex.uppercase()

// 5. 第二次 MD5（第一次结果的 ASCII 字节）→ 32 字符 HEX（大写）
val secondMd5 = MD5(firstMd5.toByteArray(Charsets.US_ASCII)).hex.uppercase()

// 6. 拼接最终密码（64 字符）
val password = firstMd5 + secondMd5
```

流程示意：

```
ZIP 文件尾部 336 字节
    │
    ├─ 验证签名: MSTCHINA 或 EPLAT
    │
    ▼
提取偏移 16-143 的 128 字节种子
    │
    ▼
第一次 MD5 → 32 字符 HEX（大写）
    │
    ▼
第二次 MD5（ASCII 编码）→ 32 字符 HEX（大写）
    │
    ▼
拼接 = 64 字符解压密码
```

## 8.3 解压

使用 zip4j 库进行带密码解压：

```kotlin
val zip4jFile = net.lingala.zip4j.ZipFile(zipFile)
zip4jFile.setPassword(password.toCharArray())
zip4jFile.extractAll(extractDir.absolutePath)
```

# 九、content.json 解析

解压后的目录中会包含 `content.json` 文件，其解析逻辑与本地模式**完全一致**，复用 `ETS100AnswerReader.parseContentJson()` 方法。

## 9.1 解析流程

```kotlin
// ReadScreen.kt - downloadAndParseHomework()

// 1. 读取 content.json
val contentJsonFile = File(extractDir, "content.json")
val json = JSONObject(contentJsonFile.readText())

// 2. 获取 structure_type
val structureType = json.optString("structure_type", "")

// 3. 调用通用解析方法
val (questions, originalContent) = ETS100AnswerReader.parseContentJson(
    json = json,
    startIndex = questionIndex,
    category = "",
    typeName = content.groupName,      // 使用作业中的题型名
    sectionCaption = content.groupName
)

// 4. 构建 Section → Paper
```

## 9.2 structure_type 与解析方式

与本地模式完全一致，详见 [答案解析方法](./ets100答案解析方法.md)。

| structure_type | 题型 | 答案位置 |
|----------------|------|----------|
| `collector.role` | 听选信息 / 回答问题 / 提问 | `info.question[].std[].value` |
| `collector.3q5a` | 3问5答 | `info.question[].std[]`（多答案） |
| `collector.choose` | 听后选择 | `info.xtlist[].answer` + `xxlist` |
| `collector.picture` | 信息转述 | `info.std[].value` |
| `collector.fill` | 听后记录/填空 | `info.std[].value` |
| `collector.dialogue` | 回答问题 | `info.question[].std[].value` |
| `collector.read` | 模仿朗读 | 无答案，跳过 |

## 9.3 文本清洗

解析答案时需清洗文本（与本地模式一致）：
- 移除 `ets_th{数字}` 前缀
- 移除 HTML 标签（`<p>`, `<br>` 等）
- 移除零宽空格（`\u200B`）
- `</p><p>` 替换为换行

## 9.4 最终数据结构

```kotlin
data class Paper(
    val paperId: Long,
    val title: String,              // 作业名称
    val dataFileName: String,       // 作业名（作为缓存 key）
    val fileSize: Long,             // 云端模式下为 0
    val sections: List<Section>,    // 题型分区列表
    val downloadTime: Long,         // 云端模式下为 0
    val regionLabel: String,        // "云端"
    val paperName: String?          // 作业名
)

data class Section(
    val caption: String,            // 题型名称
    val category: String,           // structure_type
    val typeName: String,           // 题型中文名
    val questions: List<Question>,
    val originalContent: String?
)

data class Question(
    val order: Int,                 // 全局题号
    val sectionOrder: Int,          // 分区内题号
    val sectionCaption: String,     // 分区标题
    val typeName: String,           // 题型名
    val questionText: String,       // 题目文本
    val answers: List<String>,      // 答案列表
    val originalText: String?,
    val category: String
)
```

# 十、状态管理

## 10.1 CloudHomeworkState 全局单例

```kotlin
object CloudHomeworkState {
    var homeworkList: List<HomeworkInfo>          // 云端作业列表
```

- 保存在**内存**中，Tab 切换时不丢失
- App 进程被杀后**丢失**（但下载的 ZIP 缓存文件仍保留在磁盘上）

## 10.3 清除缓存

用户可通过右下角按钮删除所有云端缓存：

```kotlin
fun clearCloudHomeworkCache() {
    // 1. 删除磁盘缓存
    File(context.cacheDir, "cloud_homework").deleteRecursively()

    downloadedHomeworkNames = emptySet()
    papers = emptyList()
    homeworkList = emptyList()
```

# 十一、UI 交互流程

## 11.1 模式选择

1. 进入 设置 → 运行授权
2. 选择"云端模式"卡片
3. 展开云端激活面板：
   - **未登录**：显示"登录 ETS100 账号"按钮 → 跳转到 `CloudActivationScreen`
   - **已登录**：显示"已激活"徽章 + 手机号（脱敏）+ "进入云端" / "退出登录"按钮

## 11.2 登录页面（CloudActivationScreen）

1. 输入手机号 + 密码
2. 点击"登录"
3. 流程：
   - 获取机器码 → 调用登录 API
   - 若 `code=30014` → 自动调用绑定 API → 绑定成功后获取父账户 ID
   - 正常登录 → 获取父账户 ID
   - 保存所有认证信息到 SharedPreferences
   - 跳回上一页，模式自动切换为 `CLOUD`

## 11.3 读取页面（ReadScreen）

1. 云端模式下，页面不自动加载，等待用户手动点击"读取"
2. 点击"读取" → 弹出**风险提示对话框**（顶号风险警告）
3. 用户确认 → 自动重新登录 → 获取作业列表 → 显示列表
4. 点击某个作业：
   - **未下载** → 开始下载 + 解压 + 解析（显示加载动画）
   - **已下载** → 跳转到试卷详情页面
   - **下载失败** → 显示失败标记，可重新点击下载
5. 试卷详情页面与本地模式完全一致

## 11.4 风险提示

云端模式在用户点击"读取"时弹出风险提示：

> **重要提示**
>
> 云端模式登录可能会导致 E听说官方客户端被退出登录（"顶号"）。
>
> - 请确认当前没有正在使用 E听说
> - 不要在考试、练习、录音提交过程中使用云端读取
> - 建议先用云端模式读取并查看答案，再去 E听说官方客户端做题

# 十三、风险与注意事项

## 13.1 顶号风险

使用云端模式登录会将 ETS100 官方客户端的登录状态踢掉，因为 ETS100 同一账号只允许一个设备在线。每次加载作业列表前的自动重新登录也会触发顶号。

**建议**：先用云端模式读取答案，再打开 E听说做题。

## 13.2 密码明文存储

登录密码以**明文**形式保存在 SharedPreferences 中，存在安全风险。这是因为需要在每次加载作业列表时自动重新登录。

## 13.3 SSL 证书校验绕过

下载 CDN 文件时跳过了 SSL 证书验证（信任所有证书），存在中间人攻击风险。原因是 CDN 的 SSL 证书配置存在问题。

## 13.4 网络依赖

云端模式完全依赖网络，无网络时无法使用。已下载并解压的作业可以离线查看（内存缓存），但进程被杀后需要重新下载。

## 13.5 缓存策略

| 缓存类型 | 存储位置 | 生命周期 |
|----------|---------|---------|
| ZIP 文件 | `cacheDir/cloud_homework/*.zip` | App 清除缓存时删除 |
| 解压目录 | `cacheDir/cloud_homework/*/` | App 清除缓存时删除 |
| 作业列表 | `CloudHomeworkState`（内存） | 进程被杀后丢失 |
| 已解析试卷 | `CloudHomeworkState.downloadedPapers`（内存） | 进程被杀后丢失 |
| 认证信息 | SharedPreferences `ets100_auth` | 手动登出时清除 |
