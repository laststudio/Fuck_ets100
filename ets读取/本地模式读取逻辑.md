# 一、本地模式概览

本地模式只读取手机本地 e听说目录中的资源，不访问 ETS100 云端接口。读取方式可以是零宽字符直读、Shizuku 或 Root，但三种授权方式拿到文件访问能力后，后续目录扫描、分组、排序和解析入口一致。

本地模式的答案内容只读取 `resource/{uuid}/content.json`，不再用 `data` 里的旧内容解析答案。`data`、assets 映射和 `res.json` 只用于定位、排序、题型标题和题号兜底。

具体题型解析规则见 [ETS100 答案解析方法](./ets100答案解析方法.md)。

# 二、数据目录结构

## 设备路径

```
/storage/emulated/0/Android/data/com.ets100.secondary/files/Download/ETS_secondary/
├── data/       # 试卷索引数据（JSON 文件，包含试卷结构和元数据）
└── resource/   # 答案内容（子文件夹，每个包含 content.json）
```

## resource 子文件夹结构

```
resource/
├── {uuid-1}/
│   ├── content.json    # 题目内容 + 答案（根据 structure_type 解析）
│   ├── res.json        # 题型配置（exam_type_list）
│   ├── ctrl.json       # 答题流程控制
│   ├── info.json       # 模板渲染数据
│   └── material/       # 音频、图片素材
├── {uuid-2}/
│   └── content.json
└── ...
```

## data 文件的两种格式

| 格式 | 特征 | 用途 |
|------|------|------|
| **常规习题** | 有 `sectionData` 字段，`newStruct: true` | 正式考试试卷，通过 `fileName` 关联 resource |
| **习题组索引** | 有 `data[]` 字段，`new_struct: 1` | 练习题库索引，答案通过扫描 resource 目录获取 |

# 三、本地读取流程

## 整体流程

```
Step 1: 扫描 resource/ 目录（优先）
    │  获取所有含 content.json 的子目录
    │  按修改时间降序排序
    │
    ▼
Step 2: 按时间分组
    │  相邻文件夹修改时间间隔 <= 3秒 -> 同一组
    │
    ▼
Step 3: 根据文件夹数量路由
    │  3 -> 广东高中解析器
    │  7 -> 广东初中解析器
    │  10 -> 北京初中解析器
    │  13 -> 北京高中解析器
    │  其他 -> 通用解析器
    │
    ▼
Step 4: 遍历每组的 content.json
    │  优先读取同组/同 resource 下的 res.json 模板顺序
    │  根据 structure_type 调用对应解析方法
    │  同类题目按模板题号或包内题号排序
    │  记录已处理的 content.json 路径（去重用）
    │
    ▼
输出结果
```

## 时间分组算法

```kotlin
fun groupByTime(folders: List<FolderItem>, thresholdMs: Long = 2000): List<List<FolderItem>> {
    val groups = mutableListOf(mutableListOf(folders[0]))
    for (i in 1 until folders.size) {
        // 使用绝对值判断时间间隔
        if (kotlin.math.abs(folders[i].lastModified - folders[i - 1].lastModified) <= thresholdMs) {
            groups.last().add(folders[i])
        } else {
            groups.add(mutableListOf(folders[i]))
        }
    }
    return groups
}
```

# 四、本地 resource 顺序判定

排序来源按以下优先级使用：

1. **优先使用同套试卷的 `res.json` 模板顺序**
   - 先读取当前 resource 分组内的 `res.json`，只接受与当前解析器地区/题型兼容的模板，避免广东试卷误用北京模板。
   - 如果当前分组没有模板，同一套试卷附近可能存在一个配置包，只包含 `res.json` / `ctrl.json` / `info.json`，没有 `content.json`；此时只允许使用修改时间接近当前分组、且地区/题型兼容的配置包。
   - 读取 `res.json.exam_type_list[]`：
     - `exam_type_collect` 对应 `content.json.structure_type`。
     - `exam_type_name` 是题型标题，例如“模仿朗读”“听选信息”“回答问题”“信息转述”“询问信息”“听后选择”“听后回答”“听后转述”“短文朗读”。
     - `exam_list[].exam_id` 或题型对象自身的 `exam_id` 形如 `{{st1}}`、`{{st2}}`，表示该 content 包在整套试卷中的模板顺序。
     - `exam_info[].value` 可给出全局题号，例如 `ets_th1=3.`、`ets_th2=4.`。
   - 读取同组所有 `content.json.info.stid`，取最小 `stid` 作为 `st1`，其余用 `stid - minStid + 1` 映射到 `{{stN}}`。
   - 匹配成功后，该 content 包按 `res.json` 的模板位置排序；题型标题和 category 也优先使用模板信息；同一个包内部再按题号排序。
   - 这适用于广东初中和北京初中的所有题型：`collector.read`、`collector.role`、`collector.picture`、`collector.choose`，也适用于其他有同类模板信息的试卷。

2. **其次使用 `data/sectionData` 作为 resource 文件夹排序兜底**
   - 只读取 `sectionData[].sectionItemData[].fileName` 建立 resource 文件夹顺序。
   - 这个步骤只用于排序，不用于解析答案内容。

3. **最后使用 resource 修改时间和包内题号兜底**
   - 如果没有 `res.json`，也没有可用的 `sectionData.fileName`，保持按时间分组得到的顺序。
   - 对同一类题目，包内题目优先按官方题号字段或 `ets_th{数字}` 排序。
   - 如果多个包都出现重复的 1、2 题号，则按 content 包顺序重新生成连续显示题号，避免显示成 `1,1,1,2,2,2`。

# 五、内置 etsresource 映射

## 目录结构

```
app/src/main/assets/etsresource/
├── beijing-C1.json, beijing-C2.json, beijing-C3.json   # 北京版初中
├── beijing-G1.json, beijing-G2.json, beijing-G3.json   # 北京版高中
├── resource-C1.json, resource-C2.json, resource-C3.json # 通用初中
└── resource-G1.json, resource-G2.json, resource-G3.json # 通用高中
```

`beijing-*.json` 和 `resource-*.json` 结构相同，仅地区不同。

## beijing-*.json 结构（试卷索引）

```json
{
  "items": [
    {
      "id": "369648",
      "name": "(新教材)BSD 7年级(上) SU1A",
      "fileIdentifiers": ["78d1654b...", "46a34280...", "..."]
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `id` | 试卷唯一标识符 |
| `name` | 试卷名称 |
| `fileIdentifiers` | 关联的 resource 子文件夹标识符数组 |

## resource-*.json 结构（资源映射表）

```json
{
  "byId": {
    "376927": {
      "name": "(新教材)7年级(上)U1A",
      "file_identifiers": ["a1dac348...", "f769c272...", "..."]
    }
  }
}
```

## 数据关联关系

```
beijing-*.json / resource-*.json
    │
    ├── items[].id / byId.keys -> paperId
    ├── items[].fileIdentifiers / byId[].file_identifiers -> resource 子文件夹名
    │
    ▼
data/{paperId} -> sectionData -> fileName -> resource/{fileName}/content.json
```

## 地区差异

| 系列 | 命名格式 | fileIdentifiers 数量 |
|------|----------|---------------------|
| C系列（初中） | `{年级}(上/下){单元}` | 约 7 个 |
| G系列（高中） | `{必修/选择性必修}{数字} {单元}` | 约 3 个（通用）/ 12-13 个（北京） |
