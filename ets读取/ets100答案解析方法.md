# 一、文档定位

本文档只说明 ETS100 题目内容的解析规则，也就是拿到 `content.json` 之后，如何根据地区、题型和 `structure_type` 提取题目、答案、选项和朗读原文。

本地文件扫描、时间分组、`res.json` 顺序判定、`data/sectionData` 兜底和 assets 映射关系，见 [本地模式读取逻辑](./本地模式读取逻辑.md)。

# 二、不同地区题型与数据结构

## 地区识别：通过 resource 文件夹数量判断

| 文件夹数 | 地区/学段 | 题型组成 |
|----------|----------|----------|
| **3** | 广东高中 | 3问5答（每题多个参考答案） |
| **7** | 广东初中 | 听选信息 + 回答问题 + 信息转述 + 提问 |
| **10** | 北京初中 | 听后选择 + 听后回答 + 听后转述 |
| **13** | 北京高中 | 听后选择 + 听后记录 + 听后转述 + 回答问题 |

## structure_type 与解析方式

| structure_type | 出现场景 | 答案位置 |
|----------------|----------|----------|
| `collector.role` | 广东初中（听选/回答/提问）、北京初中（听后回答） | `info.question[].std[].value` |
| `collector.3q5a` | 广东高中（3问5答） | `info.question[].std[]`（每题多个答案） |
| `collector.choose` | 北京初中/高中（听后选择） | `info.xtlist[].answer`，选项在 `info.xtlist[].xxlist` |
| `collector.picture` | 各地区（信息转述/听后转述） | `info.std[].value` |
| `collector.fill` | 北京高中（听后记录/填空） | `info.std[].value`，题号在 `info.std[].xth` |
| `collector.dialogue` | 北京高中（回答问题） | `info.question[].std[].value` |
| `collector.read` | 各地区（模仿朗读/短文朗读） | 无答案，展示 `info.value` 原文 |

## 各地区 content.json 数据结构

### collector.role（广东初中/北京初中）

```json
{
  "structure_type": "collector.role",
  "info": {
    "question": [
      {
        "ask": "Where is the club going to meet?",
        "askaudio": "content_1.mp3",
        "std": [{"value": "In the school library."}]
      }
    ]
  }
}
```

### collector.3q5a（广东高中）

```json
{
  "structure_type": "collector.3q5a",
  "info": {
    "question": [
      {
        "ask": "What is the man's job?",
        "std": [
          {"value": "He is a teacher."},
          {"value": "He teaches English."}
        ]
      }
    ]
  }
}
```

### collector.choose（北京初中/高中）

```json
{
  "structure_type": "collector.choose",
  "info": {
    "xtlist": [
      {
        "xt_xh": "1",
        "xt_nr": "ets_th1 What did the man forget to do?",
        "answer": "C",
        "xxlist": [
          {"xx_mc": "A", "xx_nr": "Take the bag."},
          {"xx_mc": "B", "xx_nr": "Lock the door."},
          {"xx_mc": "C", "xx_nr": "Leave food for the cat."}
        ]
      }
    ]
  }
}
```

### collector.fill（北京高中）

```json
{
  "structure_type": "collector.fill",
  "info": {
    "std": [
      {"xth": "15", "value": "答案1"},
      {"xth": "16", "value": "答案2"}
    ]
  }
}
```

### collector.dialogue（北京高中）

```json
{
  "structure_type": "collector.dialogue",
  "info": {
    "question": [
      {
        "ask": "What did you do last weekend?",
        "std": [
          {"value": "I went to the park."},
          {"value": "I visited my grandmother."}
        ]
      }
    ]
  }
}
```

### collector.picture（各地区信息转述）

```json
{
  "structure_type": "collector.picture",
  "info": {
    "std": [
      {"value": "Li Hua is a middle school student..."}
    ],
    "topic": "The Experience of Getting Sick",
    "value": "<p>Hi, I'm Li Hua...</p>"
  }
}
```

## 广东初中题型细分（同为 collector.role 的区分）

广东初中的听选信息、回答问题、提问都使用 `collector.role`。如果存在 `res.json` 模板信息，优先按 `exam_type_name` 区分题型；只有没有模板信息时，才用以下内容特征兜底判断：

| 条件 | 题型 |
|------|------|
| `ask` 包含 `<br>` 且 `askaudio` 有值 | 听选信息 |
| `askaudio` 有值（不含 `<br>`） | 回答问题 |
| `askaudio` 为空且 `ask` 包含中文 | 提问 |

# 三、各地区解析器

## 广东初中（7个文件夹）

广东初中的 `res.json` 完成顺序通常为：

| st 顺序 | 题型 | structure_type | 题号来源 |
|---------|------|----------------|----------|
| `st1` | 模仿朗读 | `collector.read` | 无答案，展示原文 |
| `st2`-`st4` | 听选信息 | `collector.role` | `exam_info[].value`，通常为 1-6 |
| `st5` | 回答问题 | `collector.role` | `exam_info[].value`，通常为 7-10 |
| `st6` | 信息转述 | `collector.picture` | `info.std[].value` |
| `st7` | 询问信息 | `collector.role` | `exam_info[].value`，通常为 1-2 |

遍历每个 content.json：
- `collector.read` -> 展示模仿朗读原文
- `collector.role` -> 遍历 `info.question[]`，优先按 `res.json.exam_type_name` 分为听选信息、回答问题、询问信息；无模板时按 askaudio/中文条件兜底
- `collector.picture` -> 从 `info.std[].value` 提取信息转述答案

输出顺序：模仿朗读 -> 听选信息 -> 回答问题 -> 信息转述 -> 询问信息

## 广东高中（3个文件夹）

遍历每个 content.json：
- `collector.read` -> 展示模仿朗读原文
- `collector.3q5a` -> 遍历 `info.question[]`，每题从 `std[]` 提取多个参考答案

输出顺序：模仿朗读 -> 3问5答

## 北京初中（10个文件夹）

遍历每个 content.json：
- `collector.choose` -> 从 `info.xtlist[]` 提取题目、选项（`xxlist`）、答案（`answer`）
- `collector.role` -> 从 `info.question[].std[].value` 提取听后回答
- `collector.picture` -> 从 `info.std[].value` 提取听后转述
- `collector.read` -> 展示短文朗读原文

输出顺序：听后选择 -> 听后回答 -> 听后转述 -> 短文朗读

## 北京高中（13个文件夹）

遍历每个 content.json：
- `collector.choose` -> 听后选择（同北京初中）
- `collector.fill` -> 从 `info.std[]` 提取填空答案（`xth` 题号 + `value` 答案）
- `collector.picture` -> 听后转述
- `collector.dialogue` -> 从 `info.question[].std[].value` 提取回答问题
- `collector.read` -> 展示短文朗读原文

输出顺序：听后选择 -> 听后记录 -> 听后转述 -> 回答问题 -> 短文朗读

## 通用解析器（文件夹数量不匹配）

遍历每个 content.json，根据 `structure_type` 分发：
- `collector.role` / `collector.3q5a` / `collector.dialogue` -> `info.question[].std[].value`
- `collector.choose` -> `info.xtlist[].answer` + `xxlist`
- `collector.fill` -> `info.std[].xth` + `value`
- `collector.picture` -> `info.std[].value`
- `collector.read` -> 展示朗读原文

# 四、文本清洗

解析答案时需清洗文本：
- 移除 `ets_th{数字}` 前缀
- 移除 HTML 标签（`<p>`, `</br>` 等）
- 移除零宽空格（`\u200B`）
- `</p><p>` 替换为换行
